Key findings and improvement opportunities

RunningMeta lead intake is active

The main workflow handles pediatric/pregnancy/office leads from the Bright Futures Facebook page, tags the lead, creates New Patient opportunities, notifies staff, sends follow-up, and routes to call control.

Watch overlapBroad Facebook lead workflow also exists

FB Multiple Opt - In Automation listens broadly for Facebook lead forms. It may overlap with the primary lead workflow and should be reviewed before adding more lead touches.

Appointment requestBookings are requests, not auto-confirmed

New Patient and Reactivation calendars are set up so appointment requests require staff confirmation. The request workflow acknowledges the request and notifies the team.

GapNo true timed appointment reminder sequence found

Google invitations and confirmation cleanup exist, but no active pre-appointment reminder cadence was found in workflow logic.

GapNo active no-show recovery found

No Show pipeline stages exist, but the audited workflows did not reference the no-show stage IDs or run a no-show nurture sequence.

GapShowed but did not start care is not specifically nurtured

A Showed tag is added when moved to Showed, but no dedicated attended/no-start sequence was found for care-plan non-starts.

RunningLost patient nurture exists

Lost new-patient opportunities are added to the broad pediatric nurture sequence after a wait. This is not the same as a specific attended/no-start campaign.

RunningManual reactivation campaign exists

The reactivation workflow starts from the dbr tag and routes positive/negative responses through the Reactivations pipeline.

Inactive draftsAI workflows are mostly draft/inactive

Closebot/customer-reply logic is published, but several AI calling and scheduling workflows are drafts and should not be treated as live automation.
